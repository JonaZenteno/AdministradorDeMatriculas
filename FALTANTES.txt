# Requerimientos del Proyecto: Sistema de Matrícula

Este documento formaliza los requerimientos pendientes y mejoras solicitadas para el sistema de matrícula escolar.

## 1. Seguridad y Control de Acceso
- **Perfil de Administrador**: Implementar un sistema de inicio de sesión (Login) al abrir la aplicación.
  - **Credenciales por defecto**: Usuario: `admin`, Contraseña: `admin`.
  - **Interfaz**: Pantalla simple con título "SISTEMA DE ADMINISTRACIÓN DE MATRICULAS ESCUELA LOS LEONES".

## 2. Gestión de Datos (Base de Datos)
- **Nuevos Campos**: Agregar los siguientes campos a la base de datos y formulario:
  - `N° de Matricula` (Identificador único adicional o administrativo).
  - `Observaciones` (Campo de texto libre).
- **Consistencia de Datos**:
  - **Mayúsculas**: Todo texto ingresado debe guardarse en MAYÚSCULAS en la base de datos.
  - **Formato de Fecha**: Estandarizar fechas al formato `dd-mm-yyyy`.

## 3. Interfaz de Usuario (UI/UX)
- **Formulario de Ingreso**:
  - **Limpieza de Formulario**: Limpiar automáticamente todos los campos después de guardar un registro exitosamente.
  - **Controles Mejorados**:
    - **Preguntas Binarias (Si/No)**: Reemplazar campos de texto por `Radio Buttons`.
    - **Parentesco**: Reemplazar campo de texto por lista desplegable (`ComboBox`) con opciones: Padre, Madre, Abuela, Tia, Hermana, Otro.
  - **Responsividad**: Asegurar que la interfaz se adapte o proporcione barras de desplazamiento (scrollbars) cuando se reduzca el tamaño de la ventana, evitando que los botones desaparezcan.

## 4. Funcionalidades del Registro
- **Edición de Matrículas**:
  - Implementar funcionalidad para modificar fichas existentes (ej. cambio de dirección, teléfonos).
  - Acceso desde la vista de "Base de Datos" (botón "Editar").

## 5. Reportes
- **PDF**: Actualizar el generador de PDF para incluir los nuevos campos y respetar el formato de los datos.
